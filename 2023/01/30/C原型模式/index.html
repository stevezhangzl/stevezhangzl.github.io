3.C++有两种声明对象的方式

有一个Class Data，用两种方式声明，然后a=b 这两种有什么区别？

3.1 第一种

声明的是自动的资源，a,b分别是对象本身，a=b 就是把b拷贝给a

3.2 第二种

是两个指针指向两块内存，a=b 后 把b的地址赋值给a 也就是a也指向b所指向的内存

C++最大的特点是支持两种声明对象的方式，而像Java只支持一种，Java中全是指针

这问题咋一看会蒙，实际上只需要明确a和b到底是什么，赋值的到底是什么就可以了，一定要注意a是对象本身还是一个地址值

深拷贝与浅拷贝

由一个错误程序引出，我们想要解码一个图片文件，生成内存对象Bitmap

ImageUtil.h

Bitmap.h

Bitmap.cpp

main.cpp

Bitmap bitmap1 = bitmap;

进行拷贝，结果程序崩溃了，在析构那里打印一下data的地址的值，结果

两个地址相同，并且析构了两次，代表同一片内存析构了两次，所以报错

为什么执行Bitmap bitmap1 = bitmap; 就会报错呢？

我们看一下内存中的情况，Bitmap在内存中就三个width height data 其中data是指针，指向malloc的内存

在进行一般的拷贝的时候，进行的是浅拷贝，也就是另一个bitmap中同样有width height 和指针的值，也就是两个bitmap中的指针指向的是同一块内存，在函数执行完，进行资源回收的时候，会执行析构，然后某一个对象会先把这一块malloc内存回收，而不通知另一个对象，另一个对象再回收的时候，是回收一片已经不存在的内存区域，所以会报错。
